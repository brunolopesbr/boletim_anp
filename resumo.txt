Resumindo, quais eram oS problemas a serem resolvidos? Os dados estavam disponíves em 88 planilhas, e cada planilha continha 18 tabelas de dados, distribuídas em 3 guias. Como foi escolhido o caminho de importar cada tabela separadamente, é preciso informar ao software usado quatro informações para cada tabela: o nome da planilha, o número da guia, a linha inicial e a linha final de cada tabela. Para importar todas as tabelas seriam necessários, então, informar mais de 5 mil parâmetros apenas para importar cada tabela.
Além da necessidade de importar um grande número de elementos, a temporalidade de cada tabela precisa ser identificada. Em cada planilha são divulgados dados mensais, e essa informação não está explicitamente informada em cada tabela. Mas, no momento em que cada tabela é retirada individualmente desse ambiente, a informação do mês de geração dos dados precisa ser anexada a cada tabela. 
Finalmente, sobra a tarefa de unir cada tabela mensal com a do mês anterior. Sendo 78 planilhas, para cada tabela são 18 operações desse tipo - mais de 1.400, dentro desse universo. É um processo que pode levar a erros e exigir limpeza de dados, se as diferentes tabelas unidas não têm o mesmo número ou nome das colunas.
Isso foi realizado: foi desenvolvido dentro do R uma função personalizada, que para cada planilha faz por duas importações de dados: na primeira importação todo o conteúdo da planilha é examinado, e identificadas as linhas que contêm os nomes das tabelas. A partir dessas informações são calculadas as linhas com o início e o final de cada tabela, para permitir a importação de cada tabela, individualmente. Finalmente, uma informação disponível nas linhas iniciais de cada planilha, com a data do arquivo, é armazenada e gravada em cada tabela, como uma nova coluna. Todos esses dataframes são armazenados em um único elemento do R, uma list.
